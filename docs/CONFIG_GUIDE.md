# Быстрые правки текста и баланса

Этот проект пока не использует внешний редактор данных. Самый удобный способ быстро менять игру - править конфиги ниже и запускать сборку.

PowerShell (Windows):

```powershell
npm.cmd run lint
npm.cmd test
npm.cmd run build
```

bash (Linux / macOS / WSL):

```bash
npm run lint
npm test
npm run build
```

CI на каждом PR запускает `npm ci → lint → test → build`, так что мелкие опечатки в строках или сломанный тип проекта вылавливаются автоматически.

## Главный баланс

Файл: `src/data/GameConfig.ts`

- `PLAYER_CONFIG` - стартовые ОЗ, атака, защита, уровень, воля.
- `LEVEL_UP_CONFIG` - опыт до уровня, прирост ОЗ, атаки, защиты и воли.
- `EXPEDITION_CONFIG` - стартовое золото, эликсиры, воля, свет фонаря.
- `COMBAT_CONFIG` - урон, защита, крит, лечение, цена навыка, элиты и боссы.
- `STRESS_CONFIG` - все изменения стресса от ударов, темноты, боссов, элит и комнат.
- `MAP_CONFIG` - глубина финала, частота боссов, веса комнат, ветвления карты.
- `ROOM_CONFIG` - награды, цены, шанс реликвий, урон ловушек, отдых, алтарь, торговец.
- `ENEMY_TIERS` - обычные враги по глубинам: ОЗ, атака, награды, профиль, кровотечение, стресс-аура.
- `BOSSES` - боссы: глубина появления, ОЗ, атака, награды, особые эффекты.

Важно: поле `name` у врагов служит ключом для перевода. Если меняешь английское имя врага, обнови такой же ключ в `src/data/EnemyTextConfig.ts`.

## Русские имена и описания врагов

Файл: `src/data/EnemyTextConfig.ts`

Здесь лежат русские названия и короткие описания карточек врагов. Это первый файл, куда стоит идти, если в бою виден английский текст.

Пример:

```ts
'Whispering Priest': {
    name: 'Шепчущий жрец',
    description: 'Каждая его фраза что-то у тебя отнимает.',
},
```

Левый ключ должен совпадать с `name` врага в `src/data/GameConfig.ts`.

## Основной UI и боевой лог

Файл: `src/systems/Localization.ts`

- Кнопки действий.
- HUD и короткие подписи.
- Боевой лог.
- Тексты комнат с наградами, уроном, покупками и смертью.

Ключи здесь - семантические (`combatBossEncounter`, `roomTreasureName`, `npcMiraPotion`, `shopBeginRun` и т.п.). Если добавляешь новую строку, придумай ей понятный namespaced ключ: `combat*` для боевого лога, `hud*` для боевого UI, `room*`/`trap*`/`rest*`/`npc*`/`shrine*` для комнат, `shop*` для финальных экранов. Механические префиксы вроде `cm_001` использовать нельзя.

Не удаляй плейсхолдеры в фигурных скобках: `{damage}`, `{value}`, `{cost}`, `{name}`, `{parts}`, `{resolve}`, `{hp}` и похожие. Код подставляет туда реальные числа.

Если строка только техническая (canvas, размеры, Z-индексы) - её место не здесь, а в `src/ui/Layout.ts` (`GAME_WIDTH`, `Depths.*` и т.п.).

## Комнаты и события

Файл: `src/systems/NarrativeManager.ts`

- Описания тайника, ловушки, привала, алтаря, торговца и пустой комнаты.
- Варианты атмосферных строк.
- Текст должен помогать понять ситуацию, но не обязан раскрывать механику как инструкция.

## Рассказчик

Файл: `src/systems/Narrator.ts`

- Короткие фразы после переходов, побед, смерти, редких событий.
- Держи стиль сухим и конкретным: предметы, звук, свет, действие.

## NPC

Файл: `src/systems/Npcs.ts`

- Имена NPC.
- Реплики.
- Услуги, цены и описания услуг.

Если меняешь цену услуги, проверь, что текст рядом не обещает другой эффект.

## Навыки

Файл: `src/systems/Skills.ts`

- Названия навыков.
- Описания.
- Числа эффектов.
- Условия и стоимость.

Для текста используй существующий формат `lt('русский', 'english')`.

## Реликвии

Файл: `src/systems/Relics.ts`

- Названия реликвий.
- Описания.
- Редкость.
- Числа эффектов.

Если меняешь эффект, сразу обнови описание, чтобы игрок видел правду.

## Стресс, добродетели и срывы

Файл: `src/systems/Stress.ts`

- Названия и описания состояний.
- Эффекты срывов и добродетелей.
- Тексты, которые появляются при изменении состояния героя.

## Мета-прогрессия

Файл: `src/systems/MetaProgressionManager.ts`

- Постоянные улучшения.
- Открытия.
- Стоимость престижа.
- Тексты прогресса между забегами.

## Проверка после правок

1. Запусти `npm run lint && npm test && npm run build` (или `npm.cmd ...` в PowerShell).
2. Открой игру: `http://127.0.0.1:5173/RpgLikeDurkest/`.
3. Проверь короткие места UI: кнопки, HUD, карточки врагов, экран смерти, экран победы.
4. Если текст длинный, лучше сократить строку, а не расширять интерфейс.
